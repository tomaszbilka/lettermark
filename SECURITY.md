# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |
| < 0.1   | No        |

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly:

1. **Preferred:** [GitHub Security Advisories](https://github.com/tomaszbilka/lettermark/security/advisories/new) (private disclosure).
2. **Alternative:** email **tomaszbilka@gmail.com** with a description and steps to reproduce.

Please do **not** open a public GitHub issue for security vulnerabilities.

## What to Expect

- **Acknowledgment** within 72 hours.
- **Initial assessment** within 7 days.
- **Fix or mitigation plan** for confirmed issues affecting supported versions.

We will coordinate disclosure timing with you and credit reporters who wish to be acknowledged (unless you prefer to remain anonymous).

## Scope

This policy covers the `lettermark` npm package published from [packages/lettermark](packages/lettermark). It does not cover third-party dependencies (the package has zero runtime dependencies).
